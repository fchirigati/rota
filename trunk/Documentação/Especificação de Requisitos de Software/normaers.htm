NORMA: ESPECIFICAÇÃO DE REQUISITOS DE SOFTWARE

 

1.      Introdução

1.1  Finalidade – Descrever a finalidade e a audiência para a qual é dirigida a ERS

1.2  Escopo – Informar o nome do software e descrever seus objetivos e benefícios

1.3  Definições, Acrônimos e Abreviaturas – Definir termos e siglas

1.4  Referências – Identificação dos documentos referenciados na ERS (titulo, data, responsável, versão, fonte)

1.5  Resumo – Descrever o que segue no resto da ERS, principalmente seu item 3.1

2.      Descrição Geral

2.1  Perspectiva do Produto – Descrever se o aplicativo é independente de outros aplicativos ou se é dependente (neste caso como se dá essa dependência) ou se é parte de outro sistema (neste caso apresentar diagrama mostrando o sistema dentro do outro sistema, e as interfaces)

2.2  Funções do Produto – Sumariar as principais funções

2.3  Características do Usuário – Descrever as características dos usuários (idade, cultura e se é usuário casual, intermitente ou permanente; esta descrição é importante para a análise da amigabilidade do sistema)

2.4  Restrições – Descrever as limitações que influam no projeto como políticas regulatórias, limitações de hardware, interfaces com outros aplicativos, considerações de segurança ou confiabilidade

2.5  Pressupostos e Dependências – Descrever os pressupostos que, no caso de não ocorrerem, podem trazer impactos ao projeto

2.6   Postergar Requisitos – Identificar os requisitos que podem ser deixados para uma futura versão do software

3.      Requisitos Específicos

3.1  Interfaces Externas

3.1.1        Interfaces dos Usuários – Modelo e descrição das interfaces dos usuários

3.1.2        Interfaces de Hardware - Interfaces com hardware, como em controle de automação de máquinas

3.1.3        Interfaces de Software – Interface com outros softwares aplicativos

3.1.4        Interfaces de Comunicação – Identificação das redes utilizadas pelo software para comunicação de dados

3.2  Requisitos Funcionais – Esta é a seção mais longa da ERS,  pois cada requisito funcional deve ser detalhadamente especificado. Essa especificação pode ser obtida usando-se uma ferramenta case para a análise

·        No caso de se utilizar a Análise Estruturada, esta seção pode ser apresentada por: DFD´s, Especificações de Processos, DER, DD e DTE

·        No caso de se utilizar a Análise OO esta seção pode ser apresentada por: Diagrama de Casos de Uso, Especificação dos Casos de Uso, Diagrama de Classes, DD, Diagramas de Atividades e Diagramas de Estado

3.3  Requisitos de Desempenho – Requisitos especiais de desempenho, como tempo de resposta ou quantidade de usuários simultâneos

3.4  Restrições de Projeto – Limitações impostas por normas, escolhas de software ou hardware, etc

3.5  Atributos – Atributos de qualidade como amigabilidade, manutenível, confiabilidade, etc. Devem ser indicados todos os atributos que se aplicam ao software.

3.6  Outros Requisitos

 

Esta norma de ERS é baseada na norma ANSI/IEEE 830 simplificada. Para cada item há o número do item, seu título, e uma pequena descrição de seu conteúdo (que não deve constar da ERS).
