NORMA: PLANO DE TESTES

 

1.      Introdução

1.1  Identificador – Identificador único para o plano de testes.

1.2  Finalidade – Descrever para quem é dirigido o plano, o título do software e o escopo do teste.

1.3  Referências – Identificar os documentos referenciados ou necessários para os testes (titulo, data, responsável, versão, fonte).

2.      Descrição geral.

2.1  Itens – Relacionar os itens ou componentes a serem testados, identificando para cada um o critério de correção ou falha.

2.2  Requisitos – Relacionar os requisitos a serem testados com seus critérios de aceitação ou falha..

2.3  Visão geral – Descrição geral das atividades de teste, técnicas, ferramentas e automação dos testes para futuras repetições.

2.4  Suspensão ou conclusão – Critérios usados para concluir ou suspender os testes e, neste último caso, como retomá-los.

2.5  Ambiente –Ambiente necessário para os testes.

2.6  Tarefas e cronograma – Tarefas e cronograma para os testes.

2.7  Riscos e gerenciamento – Principais riscos para os testes, e plano de contingência.

3.      Especificação dos testes – Divisão dos testes em vários testes, cada um com sua especificação.

3.1  Especificação 1.

3.1.1        Identificador – Identificador único desta especificação.

3.1.2        Características – Descrição dos itens a serem testados, com os requisitos associados.

3.1.3        Refinamento – Detalhamento da técnica de teste a ser utilizada, do método de analisar os resultados, das tolerâncias e dos critérios de aceitação.

3.1.4        Identificador de caso de teste – Lista dos identificadores dos casos de teste associados a esta especificação.

3.2  Especificação 2.

4.      Casos de teste.

4.1  Identificador – Identificador único do caso de teste.

4.2  Itens – Itens e requisitos a serem testados no caso de teste.

4.3  Entradas e Saídas – Especificação das entradas necessárias para o caso de teste, e das saídas esperadas.

4.4  Ambiente – Necessidades de hardware,  software e outras necessidades para o caso de teste.

5.      Procedimentos de teste – Especificação das etapas para executar um conjunto de casos de teste.

5.1  Procedimento 1.

5.1.1        Identificador.

5.1.2        Finalidade – Indicar os casos de teste incluídos neste procedimento.

5.1.3        Necessidades especiais – Necessidades especiais para a execução do procedimento.

5.1.4        Ações – Ações necessárias para executar o procedimento: iniciar, executar, medir, suspender, recomeçar, parar ou concluir.

5.1.5        Relatórios – Como serão apresentados e guardados os resultados dos testes (inclui um log).
